# 002: Internal requests

## Status
Accepted (2025-04-01)

## Context
We need to make it possible for API services to provide data from their db to other API services.

## Decision
Создать internal запросы которые возвращают всю необходимую информацию. К этим запросам могут обращаться другие сервисы например invoices-api. Internal запросы также должны быть закрыты авторизацией и аутентификацией.

## Consequences
Это решение позволит вызывать внутренние запросы в других сервисах API, что уменьшит их связность и позволит получить необходимые данные. Это также позволяет придерживаться принципам инкапсуляции и единой ответственности, т.к API должен обращаться только к своей базе данных, это позволяет недопустить путаницы в работе с базами данных.

## Alternatives
### Обращаться к базе другого сервиса напрямую
Такое решение может привести к рискам безопасности и жесткой связности. А так большой путаницы если у вас много сервисов.
